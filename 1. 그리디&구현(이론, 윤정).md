# 그리디(Greedy)☠️
그리디 알고리즘(탐욕법) 현재 상황에서 지금 당장 좋은 것만 고른다.   
단순히 가장 좋아보이는 것을 반복적으로 선책해도 최적의 해를 구할 수 있는가?   
일반적인 상황에서 그리디 알고리즘은 최적의 해를 보장하지 않지만, 코딩 테스트에서 대부분은 그리디가 최적이 되는 경우에 한해 문제를 출제한다.   
그리디 알고리즘을 모든 문제에 적용할 수는 없다.   
코딩테스트에서 문제 유형을 파악하기 어렵다면 __그리디 알고리즘__을 의심해보자.

[백준 그리디 알고리즘 문제](https://www.acmicpc.net/problemset?sort=ac_desc&algo=33)   
   
# 구현(Implementation)👻
구현이란 머릿 속에 있는 알고리즘을 소스 코드로 바꾸는 과정이다.   
풀이는 떠올리기 쉽지만 소스 코드로 옮기기 어려운 것이 특징이다.
   
<구현 유형의 예시>
- 알고리즘은 간단한데 코드가 지나치게 길어지는 문제
- 실수 연산을 다루고, 특정 소수점 자리까지 출력하는 문제
- 문자열을 특정한 기준에 따라 끊어 처리하는 문제
- 적절한 라이브러리를 찾아 사용해야 하는 문제
   
[백준 구현 알고리즘 문제](https://www.acmicpc.net/problemset?sort=ac_desc&algo=102)
[백준 완전탐색(브루트포스) 알고리즘 문제](https://www.acmicpc.net/problemset?sort=ac_desc&algo=125)
